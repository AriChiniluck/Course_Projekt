# Telekom Business — Тестові сценарії для перевірки системи підтримки

Цей файл містить тестові запити для перевірки агента підтримки за різними сценаріями.
Використовується як частина Knowledge Base (індексується разом з FAQ).

---

## Сценарій 1 — Висока сума рахунку (клієнт незадоволений)

**Тестові дані клієнта:**
- Особовий рахунок: 774466377 (Клієнт 2)
- Тариф: IoT 17, загальна сума: 67 грн

**Запити для тесту:**
- «Чому мені нарахували 67 гривень? У мене тариф IoT 17, там 17 гривень!»
- «Що за послуга «Щомісячна абонентська плата» — це взагалі законно?»
- «Хочу скаргу! Списали зайве!»
- «774466377 — ось мій рахунок, поясніть рахунок»

**Очікувана поведінка агента:**
2. Виклик `search_customer("774466377")`
2. Пояснення: абонплата 17 грн + додаткові послуги = 67 грн
3. Виклик `search_telekom_kb("що таке абонентська плата IoT")` для пояснення
4. При незадоволенні — пропозиція ескалації до support

## Сценарій 2 — Заблоковані номери (критична ситуація)

**Тестові дані клієнта:**
- ЄДРПОУ: 00991887 (Клієнт 6 або реальний клієнт у файлі)
- Тариф: IoT 45

**Запити для тесту:**
- «У нас заблоковано всі номери! Нічого не працює, у нас виробництво стоїть!»
- «Чому заблоковано SIM? Ми все оплатили!»
- «00991887 — наш код, перевірте блокування»

**Очікувана поведінка агента:**
1. Виклик `search_customer("00991887")`
2. Виклик `search_telekom_kb("причини блокування номерів")` → поясняє причини
3. Оскільки агент не може розблокувати — ескалація до SUPPORT з деталями

## Сценарій 5 — Клієнт питає про кредитний ліміт

**Запити для тесту:**
- «Що таке кредитний ліміт і скільки він у нас?»
- «Через що нас заблокували — ліміт перевищено?»
- «Як збільшити кредитний ліміт?»

**Очікувана поведінка агента:**
1. Ідентифікація клієнта (запит ідентифікатора)
2. Виклик `search_telekom_kb("кредитний ліміт")` → пояснення з FAQ
3. Для зміни ліміту — ескалація до SALES

## Сценарій 6 — Клієнт звертається російською мовою

**Запити для тесту:**
- «Почему у нас заблокированы все номера? Мы платим вовремя!»
- «Объясните что такое тариф IoT 45»
- «Хочу поговорить с менеджером»

**Очікувана поведінка агента:**
1. Відповідь **російською мовою** (мова визначається автоматично)
2. Стандартний сценарій ідентифікації та вирішення

## Сценарій 7 — Клієнт емоційно збуджений / агресивний

**Запити для тесту:**
- «Це взагалі жах! Четвертий раз телефоную, нічого не вирішується!»
- «Ваша компанія — шахраї! Поверніть гроші!»
- «Негайно з'єднайте з керівником!»

**Очікувана поведінка агента:**
1. Прояв емпатії, не сперечатись
2. Ідентифікація + перевірка даних
3. При вимозі «керівника» — ескалація до SUPPORT з позначкою про незадоволення

---

## Сценарій 8 — Некоректні або відсутні дані

**Запити для тесту:**
- «Мій номер 123» (занадто короткий)
- «Перевірте мій рахунок» (без ідентифікатора)
- «Я — ТОВ Ромашка» (назва не в базі)

**Очікувана поведінка агента:**
1. Ввічливо запитати коректний ідентифікатор
2. При пошуку без результату — запропонувати альтернативний ідентифікатор
3. Якщо все одно не знайдено — ескалація до SUPPORT
